# SSRM-3D Coupled Crisis Rollout-Window Report

## Question

Report 110 made environmental crisis repair non-substitutable. Generic environmental activity no longer counted as repair, and wrong environmental repair added damage. The designed controller still solved the task, but the learned analytic overlay failed.

This report tests the next narrower step:

```text
Can a learned plan-value head trained from short cloned simulator rollouts repair the stricter coupled crises?
```

## What Changed

The benchmark keeps the Report 110 world:

- no major crisis before the 12h development gate;
- post-gate crises require both environmental repair and social coordination;
- each crisis has one primary environmental repair class;
- wrong environmental actions add damage instead of repair credit;
- learned-controller inputs do not receive active crisis profile labels.

The new part is the plan-value target. Instead of an analytic sequence utility, each plan target is generated by cloning the current world, agent population, crisis tracker, and previous-action state, then running a short counterfactual repair window through the simulator. The target scores actual cloned outcomes: environmental progress, social progress, coupled progress, damage, trust change, resource depletion, survival, and resolution.

The learned plan-value head receives ordinary features, plan identity, recent response rates, and elapsed/remaining crisis time. It does not receive the active crisis name.

## What This Does Not Claim

This is not actor-critic, policy-gradient reinforcement learning, open-ended civilization, or subjective consciousness. It is consequence-trained plan-value supervision around an imitation-trained GRU.

It also does not prove that the agent has learned causal environmental diagnosis. The experiment is built to test that possibility and is allowed to reject it.

## Canonical Command

```bash
python3 experiments/ssrm_3d_coupled_crisis_rollout_window_controller.py --train-seeds 20260911,20260912,20260913,20260914,20260915,20260916 --tune-seeds 20261011,20261012,20261013 --eval-seeds 20261021,20261022,20261023,20261024,20261025 --hours 72 --step-hours 0.10 --population 14 --epochs 42 --hidden-size 64 --plan-epochs 72 --plan-hidden-size 72 --max-plan-examples 48000 --plan-bias-candidates 0.00,1.00,1.75,2.75,4.00,5.50,7.00,9.00,11.00 --device auto --trace-seed 20261021
```

## Result

The rollout-window plan model trained on `9248` cloned rollout examples:

```text
pairwise_accuracy = 0.524
selected_router = social_first
selected_plan_bias = 0.0
```

Validation rejected the rollout-window overlay. Higher plan biases increased some tuning coupled-response numbers, but they also increased damage or lost enough total score that the selected bias was `0.0`.

Held-out evaluation remains a failed learned result:

| Controller | Total score | Crisis score | Resolved rate | Coupled response |
|---|---:|---:|---:|---:|
| designed | `0.773` | `0.528` | `1.000` | `0.000` |
| base GRU | `0.519` | `0.000` | `0.000` | `0.000` |
| return-selected GRU | `0.518` | `0.000` | `0.100` | `0.029` |
| rollout-window GRU | `0.518` | `0.000` | `0.100` | `0.030` |
| frame MLP | `0.330` | `0.000` | `0.000` | `0.000` |
| reactive | `0.240` | `0.000` | `0.000` | `0.000` |

The rollout-window path does not improve total score, crisis score, or resolved rate over the return-selected GRU. The tiny coupled-response difference is not a substantive learned repair gain.

## Interpretation

This is a useful negative result.

The benchmark did what it was supposed to do: it replaced an analytic plan target with real cloned simulator consequences. That is closer to the field-experience direction than Report 110. But the learned overlay still does not become a reliable environmental repair controller. Validation chooses to turn the overlay off.

In plain English:

```text
short counterfactual rollout labels are not enough; the controller still needs stronger consequence-trained policy learning or explicit learned diagnosis of which repair actually matters
```

This matters for the long-term LLM-controller roadmap. A simulated field-experience critic cannot merely be trained on plausible counterfactual labels and assumed useful. It has to prove that its advice changes actions, survives held-out cases, lowers damage, and fails under targeted ablations.

## Next Step

The next benchmark should stop treating the plan-value head as a post-hoc logit overlay and train the repair policy more directly:

- actor-critic or model-based return learning inside the coupled-crisis world;
- learned active-crisis memory/state rather than only recent response rates;
- separate diagnostic heads for sanitation, route, shelter, and illness repair;
- counterfactual traces that include wrong-repair penalties and delayed damage;
- held-out crisis schedules, maps, and crisis mixes;
- an intervention test that edits the learned diagnostic state and checks whether action changes predictably.
